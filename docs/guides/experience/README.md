# `design-craft` — guides

A lightweight, portable discipline for the core design-craft loop —
**direct → systematize → structure → critique**. `aesthetic-direction` turns a
vague vibe into named, ranked goals. `design-system-foundations` derives a
token and scale taxonomy from those goals. `layout-and-information-architecture`
structures a screen or flow — hierarchy, reading flow, wayfinding. And
`design-critique` evaluates an existing surface against recognized usability
principles with severity-rated findings. A shared `quality-floor` checklist —
handle all states, the accessibility floor, "motion communicates state, honor
reduced-motion" — runs under all four.

Every skill ships portable **method**, never your stack: no UI-framework code,
no styling-language syntax, no values tables. The skills point to the
recognized standards — WCAG, the W3C Design Tokens interchange shape — and ship
the method to *derive* your values, so the discipline travels to any repo and
any toolchain.
