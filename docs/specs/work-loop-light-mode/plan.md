## Approach

Edit the `core`-pack sources, edit the Manual root `AGENTS.md`, then regenerate
and lint. Author the **risk-trigger set once** in the `work-loop` SKILL.md
source (its behavioural home), then propagate the *same wording* into the
`AGENTS.md` pair and the `CONVENTIONS.md` seed so the copies can't drift;
annotate the `new-spec` spec/plan templates' optional sections independently.

Mind the **projection asymmetry**: the work-loop skill, the new-spec templates,
and `docs/CONVENTIONS.md` (via `PROJECTED_README_OVERRIDES`) are regenerated by
`make build-self` from sources — never edit the projected copies. Root
`AGENTS.md` is the exception: it is `EXCLUDED_PATTERNS`/Manual, so `build-self`
leaves it alone; edit it **directly** and keep the adopter seed
`packs/core/seeds/AGENTS.md` in sync. The riskiest part is not code — it is
(a) prose routing, which only a judgmental adversarial pass can confirm, and
(b) projection hygiene: `build-self` must regenerate cleanly without dragging
unrelated files into the diff. No executable logic, no new artifact type — so
the testing story is goal-based gates plus the adversarial review.

## Constraints

- **ADR-0014** — the frozen design (two modes, seven-trigger set, light-mode
  contents, no-new-code vehicle). This plan implements it; it does not re-decide.
- **RFC-0025** (#237) — the accepted proposal and the gate authorising the
  CONVENTIONS/AGENTS edits.
- **Self-hosting projection** — projected artifacts edited at source + `make
  build-self`; root `AGENTS.md` edited directly (Manual) with its seed in sync.
- **Lint surfaces** — `lint-packs` (in `build-check`), `tools/lint-agent-artifacts.py`
  (projection), and `tools/lint-agents-md.py` (AGENTS hygiene); all run.
- **No new executable code / skill / artifact type**; `loop-cohort.py` and
  `lint-spec-status.py` stay byte-unchanged.

## Design (LLD)

### Design decisions
- **Single source of trigger wording.** T1 declares the canonical seven-trigger
  block in the `work-loop` SKILL.md source; it is copied verbatim into the
  `AGENTS.md` pair and the `CONVENTIONS.md` seed. The grep-equality check runs
  **after `make build-self`** and compares the *projected* SKILL.md block against
  root `AGENTS.md`, the seed, and projected `docs/CONVENTIONS.md`. Alternative
  (phrasing each independently) rejected — guarantees drift. Traces to: AC2, AC3, AC4.
- **Root `AGENTS.md` is Manual, not projected.** `build-self` excludes it
  (`self_host.py:265-266,332`), so it is edited directly and its adopter seed
  edited in parallel; do **not** expect `build-self` to propagate seed→root.
  Traces to: AC2, AC3.
- **Light mode keeps `lint-spec-status.py`, drops only the `loop-cohort` state
  machine.** The cost driver is subagent fan-out + iteration, not a no-subagent
  python lint; keeping the cheap doc-drift check preserves hygiene for ~zero
  cost. Traces to: AC6, AC7.
- **Lean spec is persisted** (`docs/specs/<feature>/spec.md`), not in-chat —
  RFC-0025 rejected the no-spec mode. Traces to: AC1, AC6.

## Rollout

- **Delivery:** big-bang via the `core` pack — light mode becomes the default for
  every adopter on next install/build. Reversible by revert (prose/seed change,
  no data migration, no published event).
- **Infrastructure:** none.
- **External-system integration:** none.
- **Deployment sequencing:** source + root-`AGENTS.md` edits → `make build-self`
  → lints, in one PR; no cross-system ordering.

## Risks

- **AGENTS.md drift:** because root `AGENTS.md` is Manual and `build-self`
  won't propagate the seed, editing one and forgetting the other silently
  diverges this repo from adopters. Mitigation: edit both in T2; grep-equality +
  `lint-agents-md.py`.
- **Projection drift:** `make build-self` may revert a projection-only edit or
  pull unrelated files into the diff (known footgun). Mitigation: edit only
  sources for projected artifacts; assert the projection diff is exactly the
  intended files.
- **Missed lint surface:** `lint-agent-artifacts.py` and `lint-agents-md.py` are
  not in `build-check`, so a green local gate can still red-CI. Mitigation: run
  both by hand in T5 (explicit ACs).
- **Trigger-wording divergence** across SKILL.md / AGENTS.md(pair) / CONVENTIONS.md.
  Mitigation: single source-of-truth wording + grep-equality check.
- **Prose routing ambiguity:** the mode branch could leave "which mode" unclear.
  Mitigation: the judgmental adversarial pass is a named construction test.
