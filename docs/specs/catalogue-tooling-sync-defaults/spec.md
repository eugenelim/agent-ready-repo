# Spec: Catalogue Tooling — Sync-Defaults

- **Status:** Draft
- **Owner:** eugenelim
- **Initiative:** ini-005 AgentBundle Portable Catalogue Tooling
- **Plan:** [`plan.md`](plan.md)
- **Constrained by:** ini-005 brief Bucket 4; [`spec/catalogue-tooling-foundation`](../catalogue-tooling-foundation/spec.md)
- **Shape:** service

> **Spec contract:** this document defines what "done" means. The implementing
> PR must match this spec, or update it. Verification must be derivable from it.

## Objective

`agentbundle/_data/install-defaults.toml` is hand-maintained today. An enterprise
fork must remember to update it manually when changing Artifactory or
adapter settings in `catalogue.toml`, creating silent drift and a source-of-truth
split.

This spec fills in `agentbundle/catalogue_tooling/defaults.py` with two commands:
`sync-defaults --check` (read-only drift detection) and `sync-defaults --write`
(atomic regeneration). It also wires `catalogue verify` to run sync-defaults
in check mode automatically when `install-defaults-output` is configured,
ensuring the generated file cannot drift silently past any verify or release gate.

The generated `install-defaults.toml` contains no credentials; only adapter
preference, Artifactory bootstrap coordinates, and default source. Auth stays
in environment variables.

## Boundaries

### Always do

- Implement `compile_defaults(config: CatalogueConfig) -> str`: pure function,
  no filesystem I/O, returns a deterministic TOML string for the given config.
- Generated TOML must include a comment header on line 1:
  `# Generated by agentbundle catalogue sync-defaults — source of truth: catalogue.toml`
- `compile_defaults` writes exactly these sections in order:
  1. `[organization]` — `preferred_adapter` (from `distribution.agentbundle.preferred-adapter`)
  2. `[organization.artifactory]` — `enabled`; when `enabled = true`, also
     `base-url`, `repository`, `bundle`, `channel`
  3. `[defaults]` — `source` (from `distribution.agentbundle.default-source`; omit
     key when `default-source` is absent)
- Implement `check_defaults(root: Path) -> SyncDefaultsResult`: read-only; returns
  `ok=True` when the current file byte-matches what `compile_defaults` would
  produce, `ok=False` otherwise (including absent file).
- Implement `write_defaults(root: Path) -> SyncDefaultsResult`: atomic replacement —
  write to `<output_path>.tmp`, then `rename` to `output_path`. Returns
  `ok=True` on success.
- Path jail: before writing, assert
  `output_path.resolve().is_relative_to(root.resolve())`; raise `CatalogueConfigError`
  on violation.
- Symlink guard: before writing, assert `not output_path.is_symlink()`; raise
  `CatalogueConfigError` on violation.
- Reuse the Artifactory field validation already in `config.py`; do not duplicate it.
- Reuse the preferred-adapter validation already in `config.py`; do not duplicate it.
- Do not preserve unknown keys from a previous generated file; full regeneration.
- Do not emit any credential fields (`token`, `password`, `key`, `secret`); validation
  already excludes them from `CatalogueConfig`.
- `agentbundle catalogue sync-defaults --root . --check` exits 0 on clean, 1 on
  drift or absent file.
- `agentbundle catalogue sync-defaults --root . --write` exits 0 on success.
- Both commands emit one JSON document to stdout conforming to `SyncDefaultsResult`
  (inherited from `CommandResult`); progress messages go to stderr.
- `catalogue verify` runs sync-defaults in check mode when `install-defaults-output`
  is present in the config; a drift finding is surfaced as a `Diagnostic` with
  severity ERROR.
- Both the repository preflight and the AgentBundle release preflight run
  `sync-defaults --check`.
- Preserve adapter precedence exactly as declared in `catalogue.toml` — emit
  `preferred_adapter` verbatim; no normalization, lowercasing, or reordering.
- Use a minimal safe TOML emitter (reuse an existing one in the codebase if present,
  or add one shared emitter to `catalogue_tooling/`; do not use a third-party lib).

### Ask first

- Changing the section order or key names of the generated TOML.
- Adding new top-level sections to the generated format.
- Making `--write` also update `catalogue.toml` (round-trip).

### Never do

- Create a second TOML format; `install-defaults.toml` is the only generated output.
- Write credentials, bearer tokens, or credential-bearing URLs to the generated file.
- Accept output paths outside the catalogue root.
- Duplicate validation logic already present in `config.py`.
- Preserve keys from a previous generated file (stale-key risk).

## Testing Strategy

- **TDD** for `compile_defaults`: one test for public config (Artifactory disabled),
  one for enterprise config (Artifactory enabled), one asserting determinism (call
  twice, assert identical output), one asserting header comment present.
- **TDD** for `check_defaults`: clean file → `ok=True`; drifted file → `ok=False`;
  absent file → `ok=False`.
- **TDD** for `write_defaults`: assert file written with correct content; assert
  atomic (tmp file gone after); assert path-jail rejection; assert symlink rejection.
- **Goal-based** for CLI: subprocess `agentbundle catalogue sync-defaults --check`
  on a clean tree exits 0; on a dirty tree exits 1. `--write` exits 0.
- **Goal-based** for verify integration: call `verify_catalogue` with config that has
  `install-defaults-output`; assert `SyncDefaultsResult` ERROR diagnostic surfaces
  when file drifted.

## Acceptance Criteria

- [ ] AC1: `compile_defaults(config)` returns a TOML string beginning with the
  generated-file header comment, containing `[organization]`, `[organization.artifactory]`,
  and `[defaults]` in that order.
- [ ] AC2: `compile_defaults` is deterministic: calling it twice with the same
  `CatalogueConfig` produces byte-identical output.
- [ ] AC3: `compile_defaults` with `artifactory.enabled = true` includes `base-url`,
  `repository`, `bundle`, `channel` under `[organization.artifactory]`.
- [ ] AC4: `compile_defaults` never emits a credential-bearing key regardless of
  config content.
- [ ] AC5: `check_defaults(root)` returns `SyncDefaultsResult(ok=True)` when the
  current `install-defaults-output` file byte-matches expected.
- [ ] AC6: `check_defaults(root)` returns `SyncDefaultsResult(ok=False)` when
  file differs or is absent.
- [ ] AC7: `write_defaults(root)` writes the output atomically: a `.tmp` file is
  created and renamed; no partial write left on failure.
- [ ] AC8: `write_defaults(root)` raises `CatalogueConfigError` when `output_path`
  resolves outside `root`.
- [ ] AC9: `write_defaults(root)` raises `CatalogueConfigError` when `output_path`
  is a symlink.
- [ ] AC10: `agentbundle catalogue sync-defaults --root . --check` exits 0 on a
  clean tree and 1 on drift.
- [ ] AC11: `agentbundle catalogue sync-defaults --root . --write` exits 0 and the
  output file matches `compile_defaults` output.
- [ ] AC12: Both commands emit valid `SyncDefaultsResult` JSON to stdout.
- [ ] AC13: All existing AgentBundle tests pass unmodified.

> **Note:** `catalogue verify` integration (surfacing a drift diagnostic when
> `install-defaults-output` is configured) is owned by spec/catalogue-tooling-verify
> step 16 (`_step_sync_defaults`). This spec exports only `check_defaults` for
> verify to call.

## Assumptions

1. Foundation spec is merged; `CatalogueConfig`, `SyncDefaultsResult`, and the
   `sync-defaults` CLI stub are all available.
2. The `install-defaults.toml` file format (section names, key names) is stable;
   the generated file supersedes the current hand-maintained file in the repo.
3. `tomllib` (stdlib 3.11) for reading; a minimal inline emitter for writing —
   no third-party TOML writer.
4. Atomic rename (`Path.rename`) is sufficient for the target filesystems; no
   `fsync` required by this spec.
5. `catalogue verify` integration is additive — it calls `check_defaults` and
   appends its diagnostics to the existing `VerifyResult`.
