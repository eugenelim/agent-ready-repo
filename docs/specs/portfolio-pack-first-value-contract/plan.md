## Declined patterns

- Tempted to hardcode the 17-pack list in the validator — declining. Glob discovery
  (`packs/*/pack.toml`) is brittle-list-free; Level B membership comes from each pack's own
  `level-b` flag. A new pack triggers enforcement automatically.
- Tempted to add a `--pack` flag for per-pack invocation — declining. The one caller
  (build-check) needs all packs in one pass; single-responsibility is served by a
  single entry point.
- Tempted to mark `first-value` required in the JSON Schema — declining. JSON Schema marks
  the section optional; the validator (not the schema) enforces it. This keeps the schema
  a type guide and the validator the enforcement point, matching the existing pattern for
  `[pack.install]` (required only via the `if/then` block, not in `required:`).
- Tempted to bump `adapter-contract` version or manually edit `plugin.json` — declining.
  `[pack.first-value]` is catalogue-internal; RFC-0011 confines adapter-contract bumps to
  projected fields. `plugin.json` is regenerated by build-self from `pack.toml`.
- Tempted to add `tutorial` as a required Level B field now — declining. The spec resolves
  this: `tutorial` is optional; the validator only enforces file existence when the field is
  declared. Pilot specs add it when they create the tutorial.

### Task 3 — Implement `tools/lint-first-value-contract.py`

**Depends on:** Task 2 (need the test stubs to drive implementation)

**Verification mode:** TDD — `python tools/test-lint-first-value-contract.py` exits 0.

**Tests:** all 22 stubs from Task 2 go green.

**Approach:**
Pure stdlib (no `agentbundle` import). CLI: `--root .` (default). Discovery: `Path(root) / "packs"` → `sorted(packs.glob("*/pack.toml"))`.

For each pack.toml:

1. Parse TOML (`tomllib`).
2. Read `fv = pack.get("pack", {}).get("first-value", None)`.
3. If `fv is None`: violation `<name>: [pack.first-value] section missing`.
4. Level A checks (always):
   - `audience-posture` present and in `{"non-technical", "mixed", "technical"}`
   - `surfaces` present, a list, ≥ 1 entry
   - `surfaces ⊆ allowed-adapters` (skip check when `allowed-adapters` absent from `[pack.install]`)
   - `prerequisites` present and a list; each entry ≤ 80 chars
   - `verification` present, string, ≤ 160 chars
   - `recovery` present, string, ≤ 300 chars
5. Level B checks (when `fv.get("level-b") is True`):
   - `starter-task` present, string, ≤ 120 chars
   - `starter-prompt` present, string, ≤ 500 chars, no `<[a-zA-Z][a-zA-Z0-9 _-]*>` token
   - `expected-result` present, string, ≤ 200 chars
   - `next-action` present, string, ≤ 120 chars
6. `writes-to-repo` check (when `fv.get("writes-to-repo") is True`):
   - `safety-gate` present, string, ≤ 200 chars
7. `tutorial` check (when `fv.get("tutorial")` is not None):
   - file exists at `Path(root) / fv["tutorial"]`

Placeholder token pattern: `_PLACEHOLDER_RE = re.compile(r"<[a-zA-Z][a-zA-Z0-9 _-]*>")`.

When `allowed-adapters` is absent from `[pack.install]`, treat as "all adapters allowed" and skip the surfaces subset check for that pack.

Exit 0 → all packs clean (print count). Exit 1 → print violations to stderr, print summary line, exit 1.
