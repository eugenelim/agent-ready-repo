## Approach

Pure additive prose + reference-file work inside the existing
`architect-diagram` skill — no code, no new dependency, no structural change.
The shape mirrors how the skill already carries its six `mermaid-*.md`
notations: three new `references/mermaid-<kind>.md` files (timeline, quadrant,
mindmap), each following the existing files' structure (skeleton, when-to-use,
pitfalls), then wire them into the three touch-points that make a notation
first-class — the intent→notation table in `notation-routing.md`, the
reference-load step and frontmatter in `SKILL.md`, and the rubric. The riskiest
part is *honesty about rendering*: these are newer Mermaid grammars, so each
must carry the `architecture-beta`-style caveat rather than being sold as a
universal default. Verification leans on the real `mmdc` binary (11.15.0) to
parse every example, plus the pack's normal gates. Metadata bump + changelog +
backlog entry close it out.

Tasks are ordered so the three reference files (T1–T3, independent of each
other) land first, then the wiring that references them (T4–T5), then the
non-content close-out (T6–T7).

## Constraints

- No ADR/RFC governs this change; it extends an existing skill within its
  established pattern. It does **not** amend the knowledge-surfaces line
  (shipped) or the pack-activation-evals rollout.
- Self-host scope: `architect`'s **skill/agent files** are user-scope-default
  and are **not** projected into this repo's working tree. But the pack's
  version still lives in `.claude-plugin/marketplace.json`, which is

  `validate` + `lint-agent-artifacts` (architect is pure-markdown — no
  pack-owned pytest) ([[project_self_host_pack_scope]],

## Design (LLD)

### Design decisions

- **Three types, not the full 14.** Timeline / quadrant / mindmap are the
  types expressible in *native Mermaid* and useful to an architect (roadmaps,
  trade-off matrices, decomposition). The editorial SVG family (pyramid, venn,
  org-chart, layer-stack) needs a non-Mermaid output path and is deferred to
  the backlog. Traces to: AC1–AC3, AC9.
- **Caveat, don't default.** Newer grammars are routed by intent but offered
  with a renderer-support caveat, reusing the skill's existing
  `architecture-beta` stance rather than inventing a second mechanism. Traces
  to: AC5.
- **Per-type budgets sharpen, don't replace.** The rubric keeps its universal
  "fits one screen" and every existing per-type section; the new work adds
  budget rows for the three new types and tightens the universal cap with
  explicit accent/edge guidance. Traces to: AC6.

### Component / module decomposition

New files (mirror existing `references/mermaid-*.md`):
`references/mermaid-timeline.md`, `references/mermaid-quadrant.md`,
`references/mermaid-mindmap.md`. Edited files: `SKILL.md`,
`references/notation-routing.md`, `references/diagram-rubric.md`,
`packs/architect/pack.toml`, `packs/architect/.claude-plugin/plugin.json`,
`.claude-plugin/marketplace.json` (regenerated by `build-self`, not
hand-edited), `docs/product/changelog.md`, `docs/backlog.md`. Traces to: all
ACs.

## Rollout

Pure skill-content + pack-metadata change. No infra, no flag, no migration.
Delivery is the merged PR; the version bump + `marketplace.json` regen is how
adopters pick it up via APM / Claude plugins / CLI. Fully reversible (revert
the PR). No deployment sequencing.

## Risks

- **Newer-grammar rendering in adopter wikis.** timeline/quadrant/mindmap may
  not render in an adopter's older Mermaid renderer. Mitigated by the
  rendering-support caveat (same posture as `architecture-beta`); not silently
  defaulted.
- **`marketplace.json` drift.** Bumping a pack's version drifts
  `.claude-plugin/marketplace.json` (all-packs aggregation); only `build-self`
  regenerates it, so it must be run and `git status` checked
  ([[feedback_nonprojected_pack_bump_drifts_marketplace]]).
- **Activation-eval near-misses.** The frontmatter `description` change could
  perturb `architect-diagram`'s Tier-A activation evals; additive keywords
  should not break existing near-miss queries, but check
  `packs/architect/.apm/skills/architect-diagram/evals/eval_queries.json`.
