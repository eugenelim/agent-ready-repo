## Blockers

**1. AC34 unimplemented — orphan-fixture detection test does not exist.** `docs/specs/skill-secrets/spec.md:632`. Spec mandates "An orphan-fixture detection check (a test that walks the fixtures tree and asserts each file's relative path appears as a substring in the test corpus) is part of the suite." Grep across `packages/agentbundle/tests/` returns zero hits for any orphan-walking test; the ROADMAP at `docs/ROADMAP.md:235` reframes AC34 as a "future test PR" inheritance invariant — the spec did not authorise that deferral. The status was nevertheless flipped to `Shipped` at `docs/specs/skill-secrets/spec.md:3`. Fix: add the orphan-walker test under `packages/agentbundle/tests/unit/test_credentials_fixtures.py` and re-check AC34, or move AC34 out of this spec via amendment before claiming Shipped.

**2. AC35 unimplemented — no-live-writes posture assertion not in the suite.** `docs/specs/skill-secrets/spec.md:640`. Spec mandates verification that no test writes to the developer's real `~/.agent-ready/`, real Keychain, or real Credential Manager. Grep finds only doc-comment references in three backend tests; no test asserts the posture. Fix: add `test_no_live_writes_posture` that walks the test corpus and asserts (a) every test setting up the dotfile/keychain uses an `isolated_home`-style fixture, or (b) the Tier-2 backends carry `SERVICE_PREFIX_OVERRIDE` / `SERVICE` monkeypatching in every keychain test — and re-check AC35.

**3. AC26 contract inversion — credentialed-skills lint blocks instead of reports.** `tools/lint-credentialed-skills.sh:250`. Spec AC26 final sentence: "Findings are *reported*, not blocked." The shell exits 1 on any finding (`if findings: sys.exit(1)`), which propagates through `conventions-check`'s `bash tools/lint-credentialed-skills.sh` invocation and turns every finding into a CI failure. `docs/CONVENTIONS.md:832` was written consistent with the *drifted* implementation, not the spec. Fix: either change `sys.exit(1)` to `sys.exit(0)` after printing findings (matching AC26), or amend AC26 to declare findings blocking and update CONVENTIONS.md / RFC-0006 accordingly. Pick one; do not leave both in disagreement.

**4. Spec `Status: Shipped` flipped while two ACs are unchecked.** `docs/specs/skill-secrets/spec.md:3`. The spec's own checklist shows `[ ]` on AC34 and AC35; the contract sentence at line 14 ("The implementing PR must match this spec, or update it") is violated by flipping Shipped with unmet criteria. Fix: revert Status to `Implementing` until findings #1 and #2 are closed, or amend the spec body to move AC34/AC35 to a follow-on with explicit ADR/RFC trail.

**5. AC22 macOS exit-code matrix unimplemented.** `packages/agentbundle/agentbundle/creds/_keychain_macos.py:41`. Spec pins a specific dispatch: `errSecInteractionNotAllowed (25308)` and `errSecNotAvailable (-25291)` → "hard fail unless `--allow-insecure-fallback` was passed"; `errSecDuplicateItem (45)` → upsert success. The backend only knows `EXIT_NOT_FOUND=44` and folds *every other* non-zero exit into a generic `Tier2HardFailError` without distinguishing locked-keychain — so the CLI cannot honor AC22's documented "Keychain locked → require `--allow-insecure-fallback`" branch. Fix: special-case `25308` and `-25291` in `_keychain_macos._classify_exit`, route them through the AC22 fallback gate in `agentbundle/commands/creds.py:run_setup`, with a parametric test pinning each code's branch.

**6. `BASE_URL` declared required by worked example violates AC29's "optional sibling".** `packs/core/.apm/skills/example-credentialed-skill/scripts/cli.py:65`. AC29 says the schema declares "one required key (`API_TOKEN`, `secret = true`) and one optional sibling (`BASE_URL`, `secret = false`)" — but `cli.py` calls `load_credentials("example", required_keys=["API_TOKEN", "BASE_URL"])`, making `BASE_URL` required at the loader level. Worse, the schema TOML has no notion of `required`/`optional` (every key in `[[namespace.keys]]` is unconditionally prompted by `agentbundle creds setup`), so "optional sibling" has no implementation surface. Fix: either (a) drop `BASE_URL` from `required_keys` in `cli.py` and have it fall through gracefully when missing (matching AC29), or (b) amend AC29 to drop "optional" and align the schema language with the implementation.

**7. Stray `packages/agentbundle/build/lib/` checked into HEAD.** `packages/agentbundle/build/lib/agentbundle/creds/loader.py` (plus 70 sibling files; verified via `git ls-files`). `build/lib/` is the standard `setuptools build` output tree; not in `.gitignore` for the package, introduced into HEAD during the spec window (not present at base commit `5671e2e`). Carries a stale shadow copy of the loader / backends / commands / shim that will diverge from `packages/agentbundle/agentbundle/` on every future edit. Fix: `git rm -r packages/agentbundle/build/lib/` and add `packages/agentbundle/build/` to `.gitignore`.

## Concerns

**8. AC26(b) AST walker silently misses obfuscation beyond the two literal-add shape.** `tools/lint-credentialed-skills.sh:168`. `_literal_string` handles `Constant(str)` and `BinOp(Add)` of two literals; a real adversarial author could use `f"{'-' * 2}token"` or `argparse.ArgumentParser.add_argument(*("--token",))`. Spec says "Deeper obfuscation (formatted strings, computed names) is out of scope per `spec.md` § Drawbacks" — but `spec.md` has no § Drawbacks section. Fix: either add a `## Drawbacks` section to the spec naming this scope limitation explicitly, or extend the AST walker to handle `JoinedStr`/`Starred(Tuple)` shapes.

**9. T7 `resolve_schema_path` returns a path that won't `is_file()`.** `packages/agentbundle/agentbundle/creds/loader.py:528`. Returns `pathlib.Path(str(parent / "references" / "creds-schema.toml"))` where `parent` is a *relative* posix-form. Docstring admits the caller is "responsible for joining the right scope root if needed", but the function name and AC24b's contract ("returns the absolute path") promise absolute. The CLI `_resolve_schema_for_namespace` works around this by joining `skill_md.parent / "references" / "creds-schema.toml"` — ignoring `resolve_schema_path` entirely. Fix: either make `resolve_schema_path` take a `scope_root: Path` and return absolute, or rename to `_relative_schema_path` and drop from `__all__`.

**10. Tier-2 backend short-circuit doesn't propagate `Tier2HardFailError` to `where`.** `packages/agentbundle/agentbundle/commands/creds.py:380`. `_tier_for_key` catches `Exception` and treats hard-fail as the key being absent — so `agentbundle creds where <ns>` prints `dotfile` or `missing` even when Tier 2 raised `ERROR_NO_SUCH_LOGON_SESSION` (AC11 mandates non-silent; Boundaries § "No silent fallback from hard-fail Win32 error codes" forbids). Fix: re-raise `Tier2HardFailError` in `_tier_for_key`; the wrapper in `run_where` already catches it (line 620).

**11. AC23 stdin-not-tty refusal exits 3 but tombstone refusal also exits 3.** `packages/agentbundle/agentbundle/commands/creds.py:80,488`. Both produce exit 3 ("other error"). A downstream log reader cannot distinguish "user piped token into stdin" (security boundary violation) from "argv parser caught tombstone" (security boundary violation) from "schema TOML failed to parse" (operator error). Fix: log a distinct stderr prefix per category (e.g. `creds setup: argv-refusal:` vs `creds setup: stdin-not-tty:`).

**12. Spec's "fixture-driven tests are the load-bearing shape" was abandoned without amendment.** `docs/specs/skill-secrets/spec.md:264`. Spec lists six required fixture files (`schema-valid/`, `schema-missing-required/`, `dotfile-valid.env`, `dotfile-quoted.env`, `dotfile-crlf.env`, `dotfile-comment.env`) — none exist on disk; tests inline content via `tmp_path` writes. Either amend the spec's Testing Strategy bullet or land the fixtures; AC34's orphan-walker reveals the inconsistency once it exists.

**13. T13c per-task ROADMAP closure conflates AC closure with task closure.** `docs/ROADMAP.md:212-234`. ROADMAP marks each T1–T13c as `[x]` and claims they "closed" specific ACs — but T8 is marked closing "AC16–AC23" while AC22 is unimplemented (finding #5), and AC26 lint inversion (finding #3) was introduced in T10 ("closed AC26, AC27"). Fix: once findings #1, #2, #3, #5 are addressed, re-walk per-task closure list.
