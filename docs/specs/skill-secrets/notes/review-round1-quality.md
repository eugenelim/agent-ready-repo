# Quality review — skill-secrets, whole-spec round 1

**Scope.** End-of-spec whole-spec pass per work-loop anti-pattern register. Integrated diff `git diff 5671e2e..8e73f3f` against `spec.md` (35 ACs; AC34, AC35 unchecked) and `plan.md` (15 tasks). Lenses: testability, observability, reliability, maintainability.

## Blockers

**1. AC34 unchecked — orphan-fixture detection test does not exist.** `docs/specs/skill-secrets/spec.md:632-639`. No test under `packages/agentbundle/tests/` walks `packages/agentbundle/tests/fixtures/creds/` and asserts each file's relpath appears as a substring in the test corpus. `ROADMAP.md` defers AC34 to "every future test PR that adds fixtures" — that is hope, not enforcement. Fix: add `packages/agentbundle/tests/integration/test_creds_fixtures_no_orphans.py` that rglobs `tests/fixtures/creds/`, rgreps each relpath under `tests/`, asserts each file is referenced; check AC34; or move AC34 to *Cross-spec / outside-the-spec-tree* with an explicit deferral RFC.

**2. AC35 unchecked — no live-writes-prevention assertion in the suite.** `docs/specs/skill-secrets/spec.md:640-644`. Spec promises an assertion that "no Keychain entry persists outside a `tmp_path`-scoped Keychain" and "target-name prefix isolation against a `tmp_path` hash". The Windows backend exposes `SERVICE_PREFIX_OVERRIDE` and exercises it (good), but no test *enforces the convention*. The macOS backend lacks the symmetric override — only takes a `monkeypatch.setattr(_keychain_macos, "SERVICE", ...)`. Future test PRs that forget the monkeypatch write to the developer's login keychain. Fix: add a `conftest.py` `autouse` fixture under `tests/integration/test_keychain_macos.py` (and Windows equivalent) asserting `SERVICE` / `SERVICE_PREFIX_OVERRIDE` is non-default at start of every test.

**3. `agentbundle creds check` silently swallows Tier-2 hard fail and reports "missing" (exit 2) instead of "Tier-2 hard fail" (exit 3).** `packages/agentbundle/agentbundle/commands/creds.py:380-389`. `_tier_for_key` wraps the Tier-2 `read_credential` call in `try / except Exception: v = None`, converting `Tier2HardFailError` into a `"missing"` tier label. `run_check` then returns exit 2 (missing) instead of AC18-mandated exit 3 (Tier-2 hard fail). Directly contradicts AC11 ("resolver does **not** fall through" on hard fail) and AC18. The integration test `test_setup_tier2_hard_fail_without_opt_out_exits_three` only exercises the *write* path; the read-path hard fail is unverified. Fix: in `_tier_for_key`, catch `Tier2HardFailError` and re-raise; narrow the broad `except Exception` to a specific catch list; add `test_check_exits_three_on_tier2_hard_fail_during_read`.

**4. `load_credentials(schema_path=...)` is a no-op parameter — the loader accepts it and never consumes it.** `packages/agentbundle/agentbundle/creds/loader.py:358-395`. AC24b states the kwarg is "for primitive authors who load the schema themselves"; the docstring at line 374 says "reserved for T7", but T7 shipped and the parameter is still ignored. Construction test `test_load_credentials_accepts_schema_path_kwarg` (test_credentials_schema.py:204-215) only inspects the function signature. Shipping a public-API parameter that has no effect is worse than not shipping it. Fix: either delete the parameter and the no-op test, or wire it: parse the schema, validate required_keys against `schema.keys`, raise `SchemaError` on mismatch.

## Concerns

**5. AC4 "mixing tiers across keys within one namespace is permitted" has no construction test.** `packages/agentbundle/tests/unit/test_credentials_loader.py:43-51` covers two keys both from env; `test_credentials_dotfile.py:173-178` covers env-wins-over-dotfile for one key. Neither exercises one key resolving at Tier 1 *and* another at Tier 2 *and* a third at Tier 3 returning a coherent `Credentials` — the cross-tier composability invariant. Fix: add `test_load_credentials_mixes_tiers_across_keys` to `test_credentials_dotfile.py` — fake Tier-2 backend resolves `BASE_URL`, env resolves `API_TOKEN`, dotfile resolves `FLAVOR`; assert all three with values from the right sources.

**6. `CredentialsMissingError` does not name which tiers were tried.** `packages/agentbundle/agentbundle/creds/loader.py:391-394`. Message reads `"namespace 'jira': missing required credential(s): API_TOKEN, BASE_URL"`. A user who ran `creds setup` cannot triage from this whether Tier 1 was skipped (typo in env-var name?), Tier 2 hit but returned empty, or Tier 3's dotfile was missing. Fix: capture which tier was checked per missing key (env name tried, Tier-2 backend label, dotfile path) and append a one-line trailer per missing key. Add `tiers_tried` to `CredentialsMissingError`'s `__init__`.

**7. macOS Tier-2 exit-code matrix from AC22 is not implemented or tested symbolically.** `packages/agentbundle/agentbundle/creds/_keychain_macos.py:60-65`. AC22's macOS row mandates symbolic recognition of `25308` (`errSecInteractionNotAllowed`) and `-25291` (`errSecNotAvailable`) with hard-fail unless `--allow-insecure-fallback` was passed. Implementation surfaces any non-44 / non-0 exit code as a generic `Tier2HardFailError(... rc={returncode}: {stderr})`. No `test_classify_macos_exit_code_*` tests mirror the Windows-side `test_classify_last_error_*` matrix. Fix: define `EXIT_INTERACTION_NOT_ALLOWED = 25308`, `EXIT_NOT_AVAILABLE = -25291`, `EXIT_DUPLICATE_ITEM = 45` as module constants; add `_classify_macos_exit_code(rc, op)` helper. (Cross-ref Adversarial Blocker #5.)

**8. Three "release-checklist" manual-QA rows have no durable artifact.** `docs/specs/skill-secrets/spec.md:262`. Spec says "Each manual-QA row carries a release-checklist line" — the three rows (Windows `getpass` real-tty refusal; `CRED_PERSIST_LOCAL_MACHINE` survives-logoff; `ERROR_NO_SUCH_LOGON_SESSION` under LocalSystem). No `RELEASE.md`, `release-checklist.md`, or equivalent exists. On the next release these will be forgotten. Fix: add `docs/product/release-checklist.md` carrying these three rows with checkboxes, or admit no release process exists and remove the "release-checklist line" promise from the spec.

**9. No test verifies Linux `creds setup` against pre-existing `~/.agent-ready/` at RFC-0004's default 0o755 is silent.** `packages/agentbundle/agentbundle/creds/loader.py:254-270`. `_ensure_parent` warns on stderr if `parent.stat().st_mode > 0o755`. The closest test (`test_existing_parent_mode_is_not_rewritten_on_posix`) chmods to `0o775` and asserts the warning fires; nothing pins the `0o755` no-warning case. A future tweak to the threshold ("> 0o700") would surface noise on every `setup` from RFC-0004-installed adopters. Fix: add `test_existing_parent_at_0o755_emits_no_warning`.

**10. Fixtures named in spec.md § Testing Strategy do not exist.** `docs/specs/skill-secrets/spec.md:266-272`. Spec inventory promises `tests/fixtures/creds/schema-valid/creds-schema.toml`, `schema-missing-required/`, `dotfile-valid.env`, `dotfile-quoted.env`, `dotfile-crlf.env`, `dotfile-comment.env` — six artifacts; none exist on disk; tests inline content via `tmp_path` heredocs. Spec over-promises. As-is, AC34's orphan check plus this entry generates a false contradiction. Fix: update spec § Testing Strategy to reflect the inline-heredoc choice.

**11. `__all__` references `EnvParseError` before the class is defined.** `packages/agentbundle/agentbundle/creds/loader.py:536-548` (`__all__`) vs `:551-557` (`EnvParseError` definition). Python's lazy binding works at runtime, but reader skimming sees `__all__` referencing names defined below. Fix: move `class EnvParseError` and `def parse_env_file` to top of module; `__all__` at the bottom.

**12. The `_tier_for_key` diagnostic helper duplicates the loader's precedence logic instead of delegating.** `packages/agentbundle/agentbundle/commands/creds.py:365-389`. Two implementations of the same precedence will drift — if the loader gains a Tier-4 (vault) or changes empty-string semantics, `_tier_for_key` won't notice. Blocker #3 is a direct consequence. Fix: refactor `load_credentials` to return a `Credentials` plus a sidecar `dict[str, str]` mapping each key to the tier it resolved at; have `_tier_for_key` call `load_credentials` and read the sidecar.

## Nits

**13. macOS backend's `SERVICE` constant lacks a `SERVICE_OVERRIDE` shape parallel to Windows.** `packages/agentbundle/agentbundle/creds/_keychain_macos.py:31`. Tests monkeypatch `SERVICE` directly; the Windows backend exposes a separate `SERVICE_PREFIX_OVERRIDE` so the production constant is immutable. Fix: add `SERVICE_OVERRIDE: str | None = None`, private `_service()` helper, update three argv-building sites; symmetry with Windows reduces cognitive load.

**14. `Credentials.__getattr__` error message doesn't list resolved keys.** `packages/agentbundle/agentbundle/creds/loader.py:121-124`. Typo'd `creds.api_token` (lowercase) gives `"namespace 'jira' has no credential 'api_token'"` without hinting the schema declared `API_TOKEN`. Fix: include resolved key list in `AttributeError` message.

**15. `_quote_for_dotfile` doesn't escape embedded double-quotes; value containing `"` round-trips to a parse error.** `packages/agentbundle/agentbundle/creds/loader.py:199-214`. Comment acknowledges "Tokens this rejects shouldn't be in scope" but there's no explicit raise. Fix: if `'"' in value or '$' in value`, raise `EnvParseError(...)` with clear message.

---

**Convergence note (round 1):** four Blockers (two overlap with Adversarial — AC34, AC35); eight Concerns; three Nits. The silent Tier-2 hard-fail swallow (Blocker #3) is the ship-bug — production behavior contradicts AC11/AC18. The `schema_path` no-op (Blocker #4) is a public-API maintainability trap. Whole-spec coverage gap: AC4 (mixing tiers), AC22 (macOS exit codes), AC34 / AC35 all carry promises without artifacts.
