## Objective

Agents and adopters running the `core` pack get a `work-loop` that **grounds its infrastructure inner loop in the platform's real contract and runtime truth** rather than in model memory or a cheap proxy. RFC-0041 gave the loop the verification *shape* for a deploy but assumed the agent knows the platform's contract and will exercise the whole chain; two independent multi-day builds on different stacks falsify both, with failures clustering into six families that are MECE along the inner-loop-stage axis (Author → Provision → Verify → Debug). Success closes all six as doctrine + reference prose:

- **(A1) Author · structural — the contract.** A new `core` skill, **`infra-contract-acquisition`**, ships a tiered, **tool-keyed (not vendor-keyed)** protocol that acquires a stack's real contract from the deterministic oracles its own toolchain ships, declares its **oracle tier + confidence**, and degrades honestly across the strong→weak spectrum — plus an **EXECUTE contract-grounding gate**: acquire before you guess.
- **(A2) Author · behavioral — the craft.** A new `operational-safety` module, **`cloud-implementation-craft`**, loaded into the **implementer's EXECUTE brief** (permissions-sufficiency, eventual-consistency / propagation, timeout / cold-start / retry / backoff, dependency ordering, terminal-failed-state, the runtime's deployment-artifact packaging / entrypoint-import model, and externalized script configuration).
- **(P) Provision — identity & convergence.** A fifth preflight artifact — **durable-credentials-once** — added to RFC-0041 P1; **terminal-failed-state** named as a convergence case; and the **reusable-script corollary** (every live-environment interaction goes through a reusable credential-reusing script, not one-off command-spray — and the scripts are **externally parameterized** so they honour organization naming conventions and stand up **like-for-like environments**).
- **(V1) Verify · static — phased oracle fidelity.** Verification oracles are phased and increasing-fidelity; the cheap-early oracle is necessary, not sufficient.
- **(V2) Verify · runtime — the data-plane probe.** A readiness-aware data-plane probe (in-network-if-private, write → read-back, poll-with-backoff, self-teardown) refining RFC-0041 P2.
- **(D) Debug — failure localization.** A **symptom→layer log playbook** strengthening the existing `observability-and-smoke` module — not a new module.
- **(Delivery) Review on the existing `quality-engineer`** loading these references in infra-flavored mode and re-deriving the contract independently — **no new reviewer / agent**; a dedicated `infra-contract-reviewer` is deferred behind an evidence trigger.

The change is delivered by editing the `core` pack's sources — a new `packs/core/.apm/skills/infra-contract-acquisition/`, a new `packs/core/.apm/skills/operational-safety/references/cloud-implementation-craft.md`, edits to that skill's `observability-and-smoke.md`, and edits to `work-loop` SKILL.md (all projected artifacts) — and adds **no executable code, no per-vendor data, and no new reviewer / agent**. It ships to every core-pack adopter.

### Always do

- Edit the **sources** under `packs/core/.apm/...`, then run `make build-self` to regenerate the projected copies under `.claude/...`. Never edit a projection directly.
- Run **all three** lint surfaces before declaring done: `make build-check` (includes `lint-packs`), `python tools/lint-agent-artifacts.py` (projection lint, not in build-check), and `python tools/lint-agents-md.py` (AGENTS.md hygiene) — the three surfaces the `infra-aware-work-loop` and `work-loop-light-mode` specs established for `core` SKILL.md / skill prose changes.
- Make `infra-contract-acquisition` **protocol-shaped**: a short protocol body (T0 detect → T1 toolchain oracle + schema slice → T2 curated platform skill, detect-and-recommend → T3 docs → Final: the runtime data-plane probe) **plus** a per-tool `references/` oracle table giving the concrete validate / plan / schema commands per stack and what each exposes.
- Make the protocol **oracle-tier-honest**: coverage is a capability spectrum keyed to the tool; the agent declares its tier + confidence; the **weak-oracle row and the runtime-probe fallback are mandatory**.
- Keep A1's EXECUTE contract-grounding gate and the P1 preflight obligation **universal across light and full mode** where RFC-0044 states so; the heavier infra-flavor layers (A2's EXECUTE load, V2's probe, the `quality-engineer` infra wiring) fire on the **existing** infra-flavored signal (the destructive/irreversible trigger + the routing-table IaC/deploy-config entry).
- Load `cloud-implementation-craft` into the **implementer's EXECUTE brief** via the orchestrator (the `operational-safety` table-routed mechanism), not subagent self-discovery — the deliberate EXECUTE-consumer extension.
- Strengthen the **existing** `observability-and-smoke` module for the debug family (D) — no new module.
- Write every protocol / module / work-loop sentence **tool-keyed and tool-neutral**; the per-tool oracle table's concrete commands are labelled the reference instance, the prose stays vendor-agnostic (Principle 1).
- State the four-way carve in the artifacts' front matter (`infra-contract-acquisition` ↔ `cloud-implementation-craft` ↔ `security-checklists` ↔ the policy-as-code/CSPM scanner).
- Bump the `core` pack version (`packs/core/pack.toml` + `packs/core/.claude-plugin/plugin.json`, with `marketplace.json` aggregated via `make build-self`), add `infra-contract-acquisition` to the catalogue/marketplace manifest, and add a `docs/product/changelog.md` `[Unreleased]` entry for the behavior change — in the **implementing PR**.

### Ask first

- **Adding, dropping, or reordering any risk trigger.** RFC-0044 adds **none** (infra already routes to full mode); if implementation seems to need one, stop and surface it.
- **Introducing any executable code, script, parser, or runner** as the feature mechanism — ADR-0034 / ADR-0031 / Decision 1 forbid it; surface before adding.
- **Adding any `operational-safety` module beyond `cloud-implementation-craft`, or any new skill beyond `infra-contract-acquisition`.**
- Expanding scope to skills other than `work-loop` / `operational-safety` / the new skill, or editing `loop-cohort.py` / `lint-spec-status.py` (frozen byte-unchanged by this spec).

### Never do

- **Ship any executable infra tooling** — no deploy wrapper, plan-parser, schema-diff, probe-runner, or oracle runner. This spec ships doctrine + prose only (ADR-0034 / ADR-0031); the loop *offers to scaffold* the P1 task-zero artifacts but does not ship them.
- **Ship a per-vendor contract / knowledge base** (Principle 1) — T2 is opt-in, detect-and-recommend content, never bundled per-vendor data.
- **Bind the protocol or any module to a specific IaC tool / vendor as normative** (Principle 1).
- **Add a fourth reviewer or a dedicated `infra-contract-reviewer`** (three-reviewer ceiling; ADR-0023) — review rides the existing `quality-engineer`; the dedicated agent is deferred behind the RFC-0044 Decision 8 evidence trigger.
- Edit a projected copy (under `.claude/...`) directly — `make build-self` reverts it; the source is the fix point.

## Testing Strategy

This change is agent-instruction prose — one new skill, one new module + one edited module, and `work-loop` SKILL.md edits — with **no executable logic** (nothing carries a compressible invariant). Verification is therefore **goal-based** plus **judgmental review** plus a **manual-QA** exercise of the new skill, with no TDD. This mirrors the strategy `infra-aware-work-loop` and `operational-safety-checklists` used for the same class of change.

- **Projection correctness** (source → projected path agree after build-self): goal-based — `make build-self` then the drift / projection gates are clean.
- **Lint conformance**: goal-based — `make build-check`, `tools/lint-agent-artifacts.py`, and `tools/lint-agents-md.py` all exit 0; `lint-packs` accepts the new skill (valid SKILL.md frontmatter + `references/`).
- **Content presence / absence** (each family's prose present; the protocol body + per-tool oracle table present; tier-honesty + weak-oracle fallback present; the EXECUTE-consumer wiring present; the four-way carve stated; *no* executable code added; `loop-cohort.py` / `lint-spec-status.py` byte-unchanged): goal-based — `grep` / `git diff` checks.
- **Doctrine correctness** (does the protocol actually *oblige* oracle-grounding before authoring and *declare tier + confidence*; is craft *loaded into the EXECUTE brief*, not only REVIEW; does `quality-engineer` *re-derive the contract independently*; is V1's phased-fidelity carve with A1 stated; is the probe *readiness-aware* and *self-tearing-down*; is tool-neutrality with the spectrum first-class): judgmental — the spec-mode and diff-mode `adversarial-reviewer` pass, since prose has no mechanical test. Because the subject crosses an infra-security boundary, a `security-reviewer` spec-stage read confirms the four-way carve keeps security config in `security-checklists`; a `quality-engineer` read confirms the `operational-safety` EXECUTE-consumer wiring is correct.
- **Manual-QA — the new skill is a user/agent-invoked artifact.** Exercise it as a user would: confirm the SKILL.md `description:` triggers on infra-authoring prompts and that the protocol body + oracle table read end-to-end as a runnable acquisition procedure (record the observation), not only that the file lints.

## Acceptance Criteria

- [x] **A1 — the `infra-contract-acquisition` skill exists and is tiered + tool-keyed.** A new `core` skill at `packs/core/.apm/skills/infra-contract-acquisition/SKILL.md` (projected to `.claude/skills/infra-contract-acquisition/`) ships a tiered protocol: **T0** detect the stack from the diff → **T1** run the toolchain oracle (`terraform validate`+`plan`, `cdk synth`, `pulumi preview`, CFN change sets, `kubectl --dry-run=server`) **plus a machine-readable schema slice** for the diff's resources → **T2** a curated platform skill for the behavioural contract no schema encodes (**detect; recommend authoring / installing one and surface it as a decision if absent** on an unfamiliar managed surface — 3-tier dependency policy) → **T3** official-docs retrieval → **Final oracle**: the runtime data-plane probe (V2).
- [x] **A1 — protocol-shaped (body + per-tool oracle table).** The skill ships **both** a short protocol body (T0→final) **and** a per-tool `references/` oracle table giving the exact validate / plan / schema commands per stack and what each exposes — so strong-tier stacks are concrete without binding the normative prose to them. Schema heterogeneity is encoded: CFN `createOnlyProperties` and Pulumi `replaceOnChanges` from schema; **Terraform force-new read from `plan` + provider docs**, since `providers schema -json` does not expose it.
- [x] **A1 — oracle-tier honesty (the generality mechanism).** Coverage is defined as a capability spectrum keyed to the tool: *strong* (Terraform / OpenTofu, Pulumi, CDK / CFN, k8s / Helm → any provider incl. Hetzner / Proxmox / vSphere / OpenStack / on-prem-k8s), *medium* (Ansible `--check --diff`, Bicep, cloud-init), *weak / none* (bespoke REST + curl, hand-rolled bare-metal). The agent **declares its tier + confidence**, and on a weak oracle **shifts weight to the runtime probe** rather than faking static coverage. The weak-oracle row and the runtime-probe fallback are present and mandatory.
- [x] **A1 — EXECUTE contract-grounding gate.** `work-loop`'s EXECUTE prose states that **before generating a CLI invocation, an IaC resource, or application code that runs on a managed runtime** (e.g. a function handler whose packaging / import model the platform dictates) **against an unfamiliar platform, the agent acquires its contract** via the protocol — never guessing a flag, schema shape, field constraint, or packaging / entrypoint assumption — named explicitly as the infra generalization of AGENTS.md's "Grep to verify a function exists before importing it." (RFC-0044 § Errata 2026-06-24.)
- [x] **A2 — `cloud-implementation-craft` module loaded at EXECUTE.** A new `packs/core/.apm/skills/operational-safety/references/cloud-implementation-craft.md` module holds the golden practices: **least-privilege-but-sufficient** permissions derived from the contract in one pass (not iterated reactively); **eventual-consistency / propagation** readiness waits; **timeout / cold-start / retry** (timeouts set to the dependency's real latency, bounded exponential backoff, **client / frontend cold-start tolerance**); **dependency ordering**; **terminal-failed-state** handling; and the **deployment-artifact packaging & entrypoint / module-resolution model** (confirm how the managed runtime packages the artifact and imports the entrypoint *before* writing imports — local package-relative or project-root assumptions that pass locally fail in a flat package root or a script-invoked entrypoint; the platform specifics defer to the T2 curated platform skill, never bundled per-vendor data); and **externalized script configuration** (infra build scripts and verify / probe / teardown scripts take resource name prefixes, region / account, tags, stage, and naming-convention tokens from **external config** — `*.tfvars` / `TF_VAR_*`, CDK context / env, a parameters or env file — **never hardcoded inline**, so a build honours organization naming + tagging conventions and the reusable harness ports across accounts to stand up **like-for-like environments**; the named mechanisms are illustrative, Principle 1). `work-loop` states the orchestrator inlines this module into the **implementer's EXECUTE brief** on infra-flavored work — the deliberate EXECUTE-consumer extension of `operational-safety` — via the `operational-safety` routing table, not subagent self-discovery. (Amended 2026-06-25 — the routing authority was relocated from a `work-loop` table to `operational-safety`'s Module index; the orchestrator-inlines-into-the-implementer-brief substance is unchanged — RFC-0044 § Errata.)
- [x] **A2 — the carve is stated.** The module's front matter names the four-way carve: `cloud-implementation-craft` owns *under-permissioning + timing / retry* (will the call path even succeed?); `security-checklists` owns *over-permissioning + security config* (is this too open?); the policy-as-code / CSPM scanner owns *config-against-policy*; `infra-contract-acquisition` owns *IaC-against-the-platform's-structural-contract*. Four distinct questions.
- [x] **P — fifth preflight artifact + terminal-failed-state.** `work-loop`'s infra multi-artifact preflight gains a **fifth artifact — a durable credential session**: establish credentials once (resolve into a static session via the toolchain's own export mechanism), assert with a single identity check, reuse for the loop, resolve profile-vs-static precedence up front — alongside RFC-0041's verify-status, teardown, test-seed, and policy-as-code / CSPM-scanner artifacts. `operational-safety`'s `state-and-idempotency` module names **terminal-failed-state** (e.g. `ROLLBACK_COMPLETE`) as a destroy-recreate convergence case.
- [x] **P — reusable-script corollary.** `work-loop` states that **every** live-environment interaction — deploy, smoke probe, **log pull, debug step** — goes through a **reusable, idempotent, credential-reusing script** that accumulates into the verify / probe / teardown harness (RFC-0041 P1's task-zero artifacts), not a stream of one-off shell commands. These scripts are **externally parameterized** — resource name prefixes, region / account, tags, stage, and naming-convention tokens live in external config (`*.tfvars` / `TF_VAR_*`, a sourced env file, CDK context / `cdk.json`), **never hardcoded inline** (the named mechanisms are illustrative, Principle 1) — so the harness honours an organization's naming + tagging conventions without editing script bodies and ports across accounts to stand up **like-for-like environments** (the property V2's ephemeral, uniquely-named probe target relies on and that `operational-safety`'s `environment-isolation` per-PR ephemeral harness builds on). It is framed as a discipline sharpening RFC-0041 P4 (agent-drives-verification), **not** a new failure family. (RFC-0044 § Errata 2026-06-24.)
- [x] **V1 — phased oracle fidelity.** `work-loop`'s infra PLAN / EXECUTE prose states verification oracles are **phased and increasing-fidelity** — static (lint / validate / policy-as-code) < plan / preview < **runtime deploy + smoke** — and the cheap-early oracle **must not be over-trusted** ("synth passed" is the local-typecheck analog, necessary not sufficient). The **A1 / V1 carve** is stated: A1 owns *did the agent consult the oracle to ground authoring before generating the resource?*; V1 owns *is a green early oracle being mistaken for "works" at verify?*.
- [x] **V2 — readiness-aware data-plane probe (refines P2).** The infra smoke gains its real shape: **in-network if private** (probed from inside the network boundary, not the operator's machine); **data-plane round-trip** (write a record → read it back; for a request path, drive it to the terminal user-visible result), not "resource exists / status healthy"; **readiness-aware** (poll until ready with bounded backoff, distinguishing not-yet-ready / propagating from broken); **self-teardown against an ephemeral, uniquely-named target** (the `defer destroy` / `dev-test-run-uuid` shape), with teardown rolled into RFC-0041 P1's teardown artifact. It **cross-links** RFC-0041 P2 and the plan template's `## Rollout` (which owns sequencing) and does **not** duplicate the GATES sequence.
- [x] **D — symptom→layer log playbook.** The existing `observability-and-smoke` module is strengthened (no new module) with the localization method: enumerate the request / deploy path's log groups up front; **map the failing status to the emitting layer** (504 / timeout → proxy / facade; 403 → authorizer / IAM; 500 → handler); **bisect the chain**; carry a **correlation id**; and **match the symptom against a failure-signature → likely-cause catalog first** (e.g. `ROLLBACK_COMPLETE` → recreate; cold-start 504 → retry / poll; conditional-write contention → contention, not an error to fix).
- [x] **Delivery — `quality-engineer` infra-flavored wiring, no new agent.** `work-loop`'s REVIEW `quality-engineer` step loads `infra-contract-acquisition` + `cloud-implementation-craft` on infra-flavored work and **re-derives the contract independently from the oracles** — never trusting the implementer's own contract evidence (which would reproduce the field-report blind spot). The prose adds **no new reviewer / agent**; it states the auth-flow-contradiction class is caught at spec stage by `design-reviewer`, and names the **dedicated `infra-contract-reviewer` as deferred behind the RFC-0044 Decision 8 evidence trigger** (a spike showing oracle-execution traffic measurably degrades `quality-engineer`'s other lenses).
- [x] **No executable mechanism.** The change adds **one** new skill directory (`infra-contract-acquisition` — protocol prose + a `references/oracle-table.md`), **one** new `operational-safety` module file (`cloud-implementation-craft.md`), and — per the maintainer-directed **progressive-disclosure restructure** (plan changelog 2026-06-24, since `work-loop` SKILL.md was already at the 1000-line body cap) — **one** new `work-loop` reference (`references/infra-verification.md`) holding the relocated infra-mode depth, plus edits to `observability-and-smoke.md`, `state-and-idempotency.md`, the `operational-safety` / `work-loop` SKILL.md files, and `implementer.md` (the consumer-side hook completing A2's EXECUTE-consumer extension) — and **no executable code** as the feature mechanism (ADR-0034 / ADR-0031 structural constraint). `loop-cohort.py` and `lint-spec-status.py` are byte-unchanged (empty `git diff` for both).
- [x] **Projection + lint.** `make build-self` regenerates the new skill, the new + edited modules, and the `work-loop` projection cleanly; `make build-check`, `python tools/lint-agent-artifacts.py`, and `python tools/lint-agents-md.py` all pass (the latter two are not in `make build-check`; run by hand / in CI).
- [x] **Release hygiene.** `packs/core/pack.toml` and `packs/core/.claude-plugin/plugin.json` versions are bumped, the projected `marketplace.json` reflects it after `make build-self` and lists `infra-contract-acquisition`, and `docs/product/changelog.md` carries an `[Unreleased]` entry for the `work-loop` behavior change.
